## 一、项目简介

在快节奏的生活中，人们常常忽视自身的情绪变化，缺乏有效的情绪觉察与疏导方式。传统日记应用只解决了"记录"问题，却无法帮助用户**理解自己、关爱自己**。

**Echo** 是一款融合 AI 智能分析与虚拟宠物陪伴的创新日记应用，致力于解决以下核心问题：

- **情绪盲区**：用户往往意识不到自己的情绪状态和变化趋势，Echo 通过 AI 自动分析日记内容，帮助用户深度理解自己的情绪
- **缺乏陪伴**：独处时的情绪无处倾诉，Echo 提供虚拟宠物与 AI 对话双重陪伴，让用户随时有"倾听者"
- **记录门槛高**：传统日记要求文字输入，Echo 支持文字、语音、照片、手写四种方式，大幅降低记录门槛
- **缺少正反馈**：记录后没有任何反馈导致坚持困难，Echo 的 AI 洞察和宠物互动为用户提供即时的情感回馈

## 二、业务价值说明

### 对用户的价值

| 维度         | 传统日记应用 | Echo                        |
| ------------ | ------------ | --------------------------- |
| **记录体验** | 单一文字输入 | 文字/语音/照片/手写多模态   |
| **情绪感知** | 无           | AI 自动分析情绪状态         |
| **情感陪伴** | 无           | 虚拟宠物 + AI 对话双重陪伴  |
| **成长洞察** | 手动回顾     | AI 自动生成四维气泡洞察     |
| **使用动力** | 纯自驱       | 宠物互动 + 成长反馈持续激励 |

### 核心业务场景

1. **日常情绪管理**：用户每天花 1-3 分钟记录心情，AI 自动分析情绪并给出温暖回应，帮助建立情绪觉察习惯
2. **压力释放出口**：在焦虑或低落时，通过 AI 对话倾诉，获得共情式的陪伴与引导
3. **个人成长追踪**：AI "成长气泡"持续发现用户身上的闪光点和进步，帮助建立积极自我认知
4. **生活事务管理**："日程气泡"从日记中智能提取待办事项，将碎片化记录转化为高效行动

## 三、AI 创新性说明（AI 在哪里、为什么是核心）

### AI 是 Echo 的核心引擎，贯穿整个产品体验

Echo 基于 **ModelScope Kimi-K2.5 大语言模型**，构建了完整的 AI 能力矩阵。AI 不是辅助功能，而是产品的**核心驱动力**——去掉 AI，Echo 将失去最核心的价值。

### AI 能力矩阵

```
日记输入 ──→ AI 情绪分析引擎 ──→ 四维气泡洞察
                │                    │
                ▼                    ▼
         AI 对话陪伴引擎        AI 日程/习惯提取
                │                    │
                ▼                    ▼
         自然语言交互            智能行动建议
```

### 四大 AI 气泡交互（核心创新点）

每篇日记经过 AI 分析后，自动生成四种维度的个性化回应气泡：

| 气泡类型      | AI 能力               | 创新点                                           |
| ------------- | --------------------- | ------------------------------------------------ |
| 💛**心情气泡** | 情绪识别 + 共情生成   | 不是冷冰冰的情绪标签，而是像朋友一样的共情式回应 |
| 🤗**拥抱气泡** | 情绪安抚 + 正向引导   | 在用户低落时主动给予温暖慰藉，而非被动等待       |
| 📅**日程气泡** | 语义理解 + 信息提取   | 从自然语言中智能提取时间、事件，无需手动输入     |
| 🌱**成长气泡** | 积极心理学 + 优势发现 | 持续发现用户进步，构建正向反馈循环               |

### AI 对话陪伴

- **流式实时响应**：采用 Server-Sent Events 技术，AI 回复逐字展示，减少等待焦虑
- **多轮上下文记忆**：支持连续对话，AI 能记住聊天历史，提供连贯的情感支持
- **人格化设定**：AI 被设定为一位温暖、善解人意的朋友，而非机器人式问答

### 为什么 AI 是核心？

| 如果去掉 AI... | 会发生什么                                   |
| -------------- | -------------------------------------------- |
| 去掉情绪分析   | 日记变成普通备忘录，失去"理解用户"的核心价值 |
| 去掉气泡洞察   | 虚拟宠物变成静态装饰，无法与日记内容产生关联 |
| 去掉 AI 对话   | 用户失去倾诉出口，产品退化为传统工具         |
| 去掉日程提取   | 需要手动管理事务，记录效率大幅下降           |

```
┌─────────────────────────────────────────────────────┐
│                AI Service Layer（核心）               │
│  ┌──────────────────┐  ┌──────────────────────────┐ │
│  │ 对话服务          │  │ 分析服务                  │ │
│  │ - 流式对话        │  │ - 气泡分析                │ │
│  │ - 上下文管理      │  │ - 情绪洞察                │ │
│  │                  │  │ - 日程/习惯提取            │ │
│  └──────────────────┘  └──────────────────────────┘ │
└─────────────────────────────────────────────────────┘
                          │
┌─────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────┘
```

### 技术选型

| 技术                     | 用途           | 选择理由                       |
| ------------------------ | -------------- | ------------------------------ |
| **SwiftUI**              | 声明式 UI 框架 | 代码简洁、预览实时、动画流畅   |
| **SwiftData**            | 数据持久化     | 原生支持、类型安全、自动迁移   |
| **PencilKit**            | 手写输入       | 系统级笔迹优化、压感支持       |
| **Speech Framework**     | 语音识别转文字 | 系统级支持、中文识别准确       |
| **ModelScope Kimi-K2.5** | AI 大模型能力  | 国产大模型、中文友好、响应迅速 |
| **SSE 流式通信**         | AI 实时响应    | 减少等待、逐字展示、体验流畅   |

### 关键技术实现

- **AI 服务化架构**：将 AI 能力拆分为 4 个独立服务（对话、气泡分析、洞察、日程习惯），职责清晰，便于独立迭代
- **流式对话**：通过 SSE（Server-Sent Events）实现 AI 回复的实时流式展示
- **多模态输入统一处理**：文字、语音转写、照片 OCR、手写识别统一转化为文本，送入 AI 分析
- **数据本地化存储**：所有日记数据通过 SwiftData 存储在本地，保障隐私安全
- **组件化 UI**：可复用组件体系（卡片、输入组件等），统一主题系统管理视觉风格

## 七、本地部署说明

### 环境要求

- **Xcode**: 15.0+
- **iOS Deployment Target**: iOS 26.0+
- **编程语言**: Swift 5.9+

### 配置 API Key

本项目使用 ModelScope Kimi-K2.5 API，需要配置 API Key 才能使用 AI 功能：

1. 在项目根目录创建 `Config.xcconfig` 文件（已被 `.gitignore` 排除）
2. 填入以下内容：

```
MODELSCOPE_API_KEY = 你的API密钥
```

3. 在 Xcode 中打开项目，选择 Target → Build Settings → 搜索 `xcconfig`，将 `Config.xcconfig` 设置为配置文件
4. 在 `Info.plist` 中添加键 `MODELSCOPE_API_KEY`，值设为 `$(MODELSCOPE_API_KEY)`

> **提示**：如果未配置 API Key，AI 相关功能将不可用，但应用的其他功能（日记记录、宠物互动等）仍可正常使用。

## 八、未来愿景

当前版本实现了 AI 日记分析与宠物陪伴的基础闭环，但 Echo 的最终目标远不止于此——我们希望让虚拟宠物真正"活"起来，成为与用户**平等共生**的数字生命。

### 🌍 宠物的独立世界 —— 创造性地利用大模型"幻觉"

> 传统观点认为大模型的"幻觉"是缺陷，但在 Echo 中，这恰恰是宠物生命力的来源。

我们计划让宠物拥有**自己的内心世界**：当用户不使用 App 时，宠物会基于大模型的创造性生成能力，在自己的世界里"生活"——它会散步、做梦、遇到新朋友、发现有趣的事物。这些经历不是预设脚本，而是由 AI 自由生成的独特叙事。

```
用户的世界                    宠物的世界
┌──────────┐               ┌──────────────┐
│ 日记记录  │──── 读取 ────→│ 理解用户生活  │
│ 情绪分享  │               │ 产生自己的    │
│ 生活事件  │               │ 联想和感悟    │
└──────────┘               └──────┬───────┘
     ▲                            │
     │                            ▼
     │                     ┌──────────────┐
     └──── 主动分享 ────── │ 宠物的冒险    │
                           │ 宠物的感悟    │
                           │ 宠物的礼物    │
                           └──────────────┘
```

**例如**：用户记录了"今天去了海边"，宠物可能在自己的世界里也展开一场海边冒险——发现了一枚奇特的贝壳，想要带回来送给用户。这个"贝壳"可能附带一段宠物自己的感悟："大海真大呀，但每一朵浪花都记得回到岸边，就像你每天都记得回来写日记一样。"

### 🔗 双向互动 —— 不再是单向的"工具使用"

当前的 AI 宠物是**被动的**：用户写日记 → AI 分析 → 宠物反馈。未来我们希望实现**真正的双向关系**：

- **宠物主动分享**：宠物会主动告诉用户"今天在我的世界里发生了什么"，分享它的冒险故事和感悟
- **宠物的独特视角**：宠物读取用户的笔记后，不只是简单总结，而是用自己的视角重新解读，产生意想不到的延伸思考
- **情感共振**：宠物的世界会与用户的情绪状态关联——用户开心时宠物的世界阳光明媚，用户低落时宠物会在自己的世界寻找"礼物"带回来安慰用户

### 🧠 RAG 长期记忆 —— 让陪伴有深度

基于 **RAG（检索增强生成）** 技术，为宠物和 AI 建立长期记忆系统：

| 能力         | 实现方式                | 用户价值                       |
| ------------ | ----------------------- | ------------------------------ |
| **记忆索引** | 将历史日记向量化存储    | AI 能回忆起用户过去的经历      |
| **智能关联** | 语义检索相关历史记录    | "上次你也遇到过类似的事..."    |
| **周期复盘** | 自动生成周/月度回顾报告 | 帮助用户发现成长轨迹与行为模式 |
| **情绪档案** | 长期跟踪情绪变化趋势    | 提供有深度的心理健康洞察       |

**最终愿景**：Echo 不再是一个"用完即走"的日记工具，而是一位拥有记忆、拥有自己世界、真正理解你的数字伙伴。每一天的记录，都在让这份陪伴变得更深。
